## Tools

List every tool your agent will use. For each tool, fill in all four fields.
You must have at least 3 tools. The three required tools are listed — add any additional tools below them.

### Tool 1: search_listings

**What it does:**
<!-- Describe what this tool does in 1–2 sentences -->
This tool will take in info about a piece of clothing the user is looking for including its description, size, and max_price and using this information it will look in `listings.json` for items that match

**Input parameters:**
<!-- List each parameter, its type, and what it represents -->
- `description` (str): A short description of the clothing piece 
- `size` (str): The size of the piece the user is looking for
- `max_price` (float): A maximum price the user is willing to pay

**What it returns:**
<!-- Describe the return value — what fields does a result contain? -->
Returns a list matching listings, sorted by relevance (best match first).

**What happens if it fails or returns nothing:**
<!-- What should the agent do if no listings match? -->
If if nothing matches, returns an empty list.

### Tool 2: suggest_outfit

**What it does:**
<!-- Describe what this tool does in 1–2 sentences -->
This tool will take in an item that was returned by Tool 1 `search_listings` and the user's current wardrobe and use the Groq API to suggest a cohesive, stylized outfit recommendation.

**Input parameters:**
<!-- List each parameter, its type, and what it represents -->
- `new_item` (dict): The new item received from Tool 1 that the user would consider buying
- `wardrobe` (dict): The user's items in their current wardrobe

**What it returns:**
<!-- Describe the return value -->
Returns a string generated by Groq suggesting what item to pair the new item with from the user's existing closet, including brief styling tips.

**What happens if it fails or returns nothing:**
<!-- What should the agent do if the wardrobe is empty or no outfit can be suggested? -->
If the wardrobe is empty then return styling advice for the item instead.

**What it does:**

**What happens if it fails or returns nothing:**
<!-- What should the agent do if the outfit data is incomplete? -->

## Planning Loop

**How does your agent decide which tool to call next?**
<!-- Describe the logic your planning loop uses. What does it look at? What conditions change its behavior? How does it know when it's done? -->
1. **Initialization** 
     - Start by calling `new_session(query, wardrobe)` to create the session dictionary.

2. **Parsing Phase**
     - Extract `description`, `size`, and `max_price` from `session["query"]`.
     - Store these in `session["parsed"]` (e.g., as `{"description": "...", "size": "...", "max_price": ...}`).
     
3. **Search Phase (Tool 1)**
     - Call `search_listings(description=session["parsed"]["description"], size=session["parsed"]["size"], max_price=session["parsed"]["max_price"])`.
     - Store the returned list of dictionaries in `session["search_results"]`.

4. **Condition Check 1 (Search Results Check):**
     - IF `session["search_results"]` is empty (length if 0):
          - Set `session["error"]` to "No matching listings."
          - Return `session` immediately and terminate the loop.
     - ELSE (results exist):
          - Set `session["selected_item"] = session["search_results"][0]` (the top/first listing)
          - Proceed to the next step.

5. **Outfit Phase (Tool 2):**
     - Call `suggest_outfit(new_item=session["selected_item"], wardrobe=session["wardrobe"])`.
     - Store the returned string in `session["outfit_suggestion"]`.

6. **Condition Check 2 (Outfit Recommendation Check):**
     - IF `session["outfit_suggestion"]` is empty or `None`:
          - Set `session["error"]` to "Failed to generate outfit suggestion."
          - Return `session` immediately and terminate the loop.
     - ELSE:
          - Proceed to the next step.

7. **Fit Card Phase (Tool 3):**
     - Call `create_fit_card(outfit=session["outfit_suggestion"], new_item=session["selected_item"])`.

8. **Condition Check 3 (Fit Card Check):**
     - IF `session["fit_card"]` is empty or `None`:
          - Set `session["error"]` to "Failed to generate fit card."
          - Return `session` immediately.
     - ELSE:
          - Return `session` (success!).

## State Management

**How does information from one tool get passed to the next?**
<!-- Describe how your agent stores and accesses state within a session. What data is tracked? How is it passed between tool calls? -->
The agent manages state using a centralized mutable `session` dictionary created at initialization.

Data is propagated through the workflow sequentially using these tracked keys:
- `session["query"]`: Stores the original user text string.
- `session["wardrobe"]`: Stores the user's initial closet items dictionary.
- `session["parsed"]`: Tracks structural constraints extracted from raw query text.
- `session["search_results"]`: Tracks lists of items matched by Tool 1.
- `session["selected_item"]`: Points specifically to index 0 of our search results to feed Tool 2.
- `session["outfit_suggestion"]`: Captures the text string returned by Tool 2.
- `session["fit_card"]`: Stores the final social media text block returned by Tool 3.
- `session["error"]`: Tracks any early termination error messages.

## Error Handling

For each tool, describe the specific failure mode you're handling and what the agent does in response.

| Tool | Failure mode | Agent response |
|------|--------------|----------------|
| search_listings | No results match the query | Stops loop execution early, saves 'No matching listings.' into `session["error"]`, and exits back to user. |
| suggest_outfit | Wardrobe is empty | Stops loop execution early, saves 'Failed to generate outfit suggestion.' into `session["error"]`, and exits back to user. |
| create_fit_card | Outfit input is missing or incomplete | Stops loop execution early, saves 'Failed to generate fit card.' into `session["error"]`, and exits back to user. |
